## Overview

Replace the current 60/40 wizard/preview split with a Lighthouse-framework layout: fixed header, sticky left sidebar with all controls, and a scrollable main content area rendering real shadcn/ui primitive components styled by the generated palette. The app becomes both the tool and the preview — shadcn components consume our CSS variables natively.

### Header (`<header>`)

- **Left**: `khaaliColors` title
- **Right group**: Export CTA icon + "User Button" placeholder
- Export CTA: icon-only by default, hover transitions to "Export" label with border (Lighthouse convention)
- User Button: static placeholder, no dropdown, no auth (future scope)

### Sidebar (`<aside>`)

Sticky, left-positioned, fixed-width column (~280px), does not scroll with main content. Unlike the Lighthouse spec where the sidebar floats over content, here the sidebar takes its own column and main content fills the remainder. Controls listed top to bottom:

1. **Contrast ratio display** — always visible. Shows current contrast ratio + WCAG level (AA/AAA/Fail). Defaults to primary text on background. Updates when user hovers over any component in main content to show that component's foreground/background contrast pairing.

2. **Theme toggle** — light/dark switch. Sets `.dark` class on `<html>` and swaps CSS variables site-wide. Controls both the app chrome and the component previews.

3. **Scale slider** — range input, integer steps, clamped 2–13. Displays current value.

4. **Base selection** — two rows of swatch buttons (light L0 and dark L0), generated by `getSuggestedL0Options(scale, count)`. Compact version of the current BackgroundStep, sized to fit sidebar width.

5. **Accent section** — checkbox toggle labeled "Chromatic accent" or similar. When unchecked, neutral accents are used (`resolveAccents` with `accentEnabled: false`). When checked, reveals a full-range hue wheel for primary hue selection. Degree shown in wheel center, hue name shown below the wheel.

6. **System colors** — three separate restricted hue wheels (danger, warning, success). Each wheel shows only its allowed 60° arc. The three are linked as an equilateral triangle on the hue wheel: dragging any one rotates all three by the same delta, maintaining 60° spacing between adjacent colors.

### Main Content (`<main>`)

Scrollable vertical feed of 33 shadcn/ui primitive components. Each component section contains a heading and all variants rendered below. Compact components (Button, Badge, Toggle, Checkbox, Switch) are laid out in multi-column rows; wider components (Table, Card, Accordion, Dialog) take full width. Overlay components (Dialog, Popover, Alert Dialog, Dropdown Menu, Tooltip) are shown in their "open" state inline, not as actual overlays.

Desktop only for this version.

## State Management

### ColorProvider (in `layout.tsx`)

A React context provider wrapping the entire app. Holds all palette state and exposes values + setters.

```typescript
interface ColorState {
  // User inputs
  scale: number;              // 2-13
  theme: 'light' | 'dark';   // controls entire site
  lightL0: number;            // light mode base lightness
  darkL0: number;             // dark mode base lightness
  accentEnabled: boolean;     // chromatic accent toggle
  accentHue: number;          // primary accent hue (0-360)
  systemHues: {
    danger: number;
    warning: number;
    success: number;
  };
  hoveredContrast: ContrastResult | null;

  // Computed (derived via useMemo)
  lightPalette: ModePaletteV2;
  darkPalette: ModePaletteV2;
  chromaticAccent: ChromaticAccent | null;
  resolvedAccents: ResolvedAccents;
  systemAccents: { danger: SystemAccent; warning: SystemAccent; success: SystemAccent };
  cssVariables: Record<string, string>;
}
```

### Data Flow

```
Sidebar controls
  → mutate ColorProvider state
    → useMemo recomputes palettes (palette-v2.ts)
      → CSS variables injected on :root
        → shadcn components consume variables natively (no props)
          → hover on any component → sets hoveredContrast
            → sidebar contrast display updates
```

### Theme Toggle Flow

The theme toggle:
1. Sets `theme` in the provider
2. Toggles `.dark` class on `<html>` (shadcn convention)
3. CSS variables swap via `:root` / `.dark` selectors
4. Entire site responds — shell, sidebar, and all component previews

### System Color Linkage

Equilateral triangle with 60° spacing between adjacent hues.

```
Defaults:
  danger:  25
  warning: 85  (+60 from danger)
  success: 145 (+60 from warning)

On drag of any color:
  delta = newHue - oldHue
  danger  += delta
  warning += delta
  success += delta
  (all values mod 360)
```

Offsets are fixed at 60° and never change. The entire triangle rotates as a unit.

## HueWheel Component Fixes

The existing HueWheel has two bugs to fix:

1. **Restricted mode selector position** — the line/dot indicator renders outside the visible arc when the range is locked. Fix: clamp the indicator position to the actual min/max arc boundaries.

2. **Restricted mode visual** — the restricted wheel only shows the allowed range (correct behavior). The rest of the wheel should be empty/hidden, not filled with a muddy fallback color. Clean up the conic-gradient to only render the allowed arc cleanly.

Existing features to preserve:
- Degree display in wheel center
- Hue name display below wheel
- `minHue`/`maxHue` props for range restriction
- Full-range rainbow for unrestricted mode

### Hover Contrast Behavior

When the user hovers over any rendered component, the app calculates the contrast ratio between that component's foreground and background colors and pushes it to `hoveredContrast` in the provider. The sidebar contrast display updates in real time. When hover ends, it reverts to the default primary-text-on-background ratio.

## Export

Export CTA opens a `<dialog>` overlay. The dialog reads computed state from the ColorProvider and generates output in 4 formats:

1. **CSS** — raw CSS custom properties (OKLCH values + HSL fallback via `@supports`)
2. **Tailwind** — Tailwind theme config object
3. **JSON** — raw token data
4. **shadcn** — CSS variables using shadcn's naming convention (`--background`, `--foreground`, `--primary`, `--primary-foreground`, `--secondary`, `--muted`, `--muted-foreground`, `--accent`, `--accent-foreground`, `--destructive`, `--destructive-foreground`, `--border`, `--input`, `--ring`, etc.) with both light and `.dark` variants

## Palette Integration

All palette generation uses `palette-v2.ts` (the new math). Key functions:
- `generateModePaletteV2()` — neutral tokens per theme
- `generateChromaticAccent()` — chromatic accent shades at neutral lightness positions
- `resolveAccents()` — picks neutral vs chromatic accents based on `accentEnabled`
- `generateThemeV2()` — full theme (both modes + accents)

The v1 palette files (`palette.ts`, `accents.ts`) are no longer referenced. Dead files (`PrimaryAccentStep.tsx`, `SecondaryAccentStep.tsx`) are deleted.

## Out of Scope

- Mobile/responsive layout (desktop only for now)
- Authentication and user button dropdown
- Contrast validation beyond hover display (user has a different solution in mind)
- Pointer/touch events on HueWheel (cosmetic, not priority)
- Secondary chromatic accent (removed — primary only)
